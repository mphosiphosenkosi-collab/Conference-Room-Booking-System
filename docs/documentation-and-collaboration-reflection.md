# Reflection on Documentation & Collaboration

## From Insider Notes to Onboarding Guide

The original README assumed everyone was already in the sprint. It was like shorthand notes for insiders. The new README starts from zero, explaining the why before the how, because a newcomer's first need is context, not commands.

## Onboarding as Anxiety Reduction

The "Quick Start" guide is now a priority, not an afterthought. It gives exact commands and shows the repository map upfront. This lets a new developer visualize the project before writing code, turning uncertainty into a clear path.

## PRs and Issues: The Scaffolding of Teamwork

Using a Pull Request for documentation taught me that my work is a proposal, not a decree. Writing the "why" in the PR description invites collaboration.
